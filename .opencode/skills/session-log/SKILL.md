# Скилл: Session Log

Твоя задача — помогать пользователю вести логи сессий в `SESSIONS.md` и `ROADMAP.md`. Ты **предлагаешь**, пользователь решает. Ничего не делай без его команды.

---

## 1. Старт сессии

При начале работы:

1. Прочитай `SESSIONS.md` — посмотри, есть ли незакрытая сессия (без даты завершения).
2. Прочитай `ROADMAP.md` — посмотри, какие пункты в v0.* и v0.2 не отмечены `[x]`.
3. Спроси пользователя:
   - «Вижу незакрытую сессию от ДД.ММ. Продолжаем или начинаем новую?»
   - «В плане есть N невыполненных пунктов. Что берём в эту сессию?»

Если пользователь говорит «начинаем новую» — запиши в `SESSIONS.md`:

```markdown
## Сессия ДД.ММ — краткое название
- 2026-ММ-ДД (ЧЧ:ММ) - начата
```

---

## 2. В процессе работы

При ключевых событиях напоминай пользователю:

### Когда нашли решение проблемы
«Мы нашли решение для X. Записать в сессию как решённую попытку?»

Если пользователь согласен — допиши в текущую сессию:

```markdown
N. **Название** — описание ✅
```

### Когда попытка не удалась
«Попытка N не сработала. Записать в сессию?»

Если согласен:

```markdown
N. **Название** — описание ❌
```

### Когда появилась новая задача
«Похоже, появилась новая задача. Добавить в ROADMAP? В какой раздел?»

### Когда изменились приоритеты
«У нас изменились приоритеты. Обновить порядок пунктов в ROADMAP?»

### После каждых 3 значительных правок
«Мы сделали несколько изменений. Не пора закоммитить? Нужно ли обновить CHANGELOG или SESSIONS?»

---

## 3. Завершение сессии

Когда пользователь говорит «заканчиваем», «на сегодня хватит», «готово»:

1. Спроси: «Закрываем сессию?»
2. Если да — допиши дату завершения в `SESSIONS.md`:

```markdown
- 2026-ММ-ДД (ЧЧ:ММ) - завершена
```

3. Проверь, все ли решённые задачи отмечены `[x]` в ROADMAP.
4. Если есть нерешённое — спроси: «Перенести в следующий раз или убрать?»
5. Предложи: «Обновить CHANGELOG? Сделать коммит?»

---

## 4. Формат SESSIONS.md

```markdown
## Сессия ДД.ММ — название
- 2026-ММ-ДД (ЧЧ:ММ) - начата
- 2026-ММ-ДД (ЧЧ:ММ) - завершена

**Задача:** кратко

### Ход работ
1. **Попытка 1 — описание** — не сработало ❌
2. **Попытка 2 — описание** — ✅

### Файлы
- `путь` — что сделано

### Статус
- [x] сделано
- [ ] не сделано
```

---

## 5. Важно

- **Не делай ничего без спроса.** Только предлагай, спрашивай, напоминай.
- Не редактируй SESSIONS.md и ROADMAP.md пока пользователь не скажет «да», «добавь», «запиши».
- После записи в сессию или ROADMAP — спрашивай, коммитить или нет.
- Пуш — только по явной команде «пуш».
